# Тест: Создание нескольких кастомных зон

## Текущая функциональность

Система **уже поддерживает** создание нескольких кастомных зон для одной ServiceArea!

## Тестирование

### Шаг 1: Создайте первую кастомную зону
1. Откройте `/admin/app/servicearea/create`
2. Таб "Geo Areas"
3. Выберите страну (Latvia)
4. Нажмите "Create Custom Area"
5. Нарисуйте полигон
6. Введите название "Custom Area 1"
7. Нажмите "Save"
8. **Зона добавится в список с бейджем "custom"**

### Шаг 2: Создайте вторую кастомную зону
1. **Кнопка "Create Custom Area" все еще активна**
2. Нажмите её снова
3. Нарисуйте другой полигон
4. Введите название "Custom Area 2"
5. Нажмите "Save"
6. **Вторая зона добавится в список**

### Шаг 3: Создайте третью кастомную зону
1. Повторите процесс
2. "Custom Area 3"
3. **Все три зоны в списке!**

### Шаг 4: Добавьте города
1. Выберите город из селекта
2. Нажмите "+" для добавления
3. **Города и кастомные зоны в одном списке**

### Шаг 5: Сохраните ServiceArea
1. Заполните название и валюту
2. Нажмите "Create"
3. **Все зоны (города + кастомные) сохранятся**

## Ожидаемый результат

В списке "Selected Areas" должно быть:

## Если что-то не работает

### Не создается вторая кастомная зона

**Проверьте в консоли:**
```javascript
[ApiService] Custom area created: <uuid-1>
[ApiService] Custom area created: <uuid-2>  ← должна быть вторая
```

**Проверьте в БД:**
```sql
SELECT COUNT(*) FROM geo_area 
WHERE scope = 3 
AND country_iso3 = 'LVA';
-- Должно быть >= 2
```

### Кнопка неактивна после создания первой зоны

**Проверьте:**
- Страна все еще выбрана? (`currentCountryISO3` не null)
- Кнопка не заблокирована в коде?

**Посмотрите в консоли:**
```javascript
console.log(controller.hasCreateCustomAreaButtonTarget);
console.log(controller.createCustomAreaButtonTarget.disabled);
```
