# Интеграция гексагональной сетки H3 в ServiceArea

## Обзор

В проект успешно интегрирована библиотека [H3 (Uber's Hexagonal Hierarchical Spatial Index)](https://h3geo.org/) для отображения гексагональной сетки на карте в разделе ServiceArea.

## Где используется

Гексагональная сетка H3 интегрирована в **два места**:

1. **Основная карта ServiceArea** (`geo_area_map_controller.js`) - для выбора существующих гео-зон
2. **Модальное окно создания кастомных зон** (`custom_area_modal_controller.js`) - для рисования новых полигонов

## Архитектура решения

Решение реализовано с соблюдением принципов **ООП** и **SOLID**:

### 1. HexagonService (Single Responsibility, Open/Closed)

**Расположение**: `assets/services/HexagonService.js`

**Ответственность**: Работа с H3 гексагональной индексацией.

**Основные методы**:
- `setResolution(resolution)` - установка разрешения H3 (0-15)
- `latLngToH3(lat, lng, resolution)` - конвертация координат в H3 индекс
- `getHexagonBoundary(h3Index)` - получение границ гексагона
- `getHexagonsForMapBounds(mapBounds, resolution)` - генерация гексагонов для видимой области карты
- `hexagonsToGeoJson(h3Indexes)` - конвертация H3 индексов в GeoJSON
- `HexagonService.getResolutionInfo(resolution)` - получение информации о разрешении

**Принципы SOLID**:
- **SRP**: Отвечает только за работу с H3
- **OCP**: Расширяем через конфигурацию без изменения кода
- **LSP**: Может быть заменен другой реализацией гексагональной индексации
- **ISP**: Предоставляет только необходимые методы
- **DIP**: Не зависит от конкретных реализаций карты

### 2. MapService (Open/Closed Principle)

**Расположение**: `assets/services/MapService.js`

**Расширение**: Добавлены новые методы для работы с гексагональными слоями.

**Новые методы**:
- `addHexagonLayer(layerId, geoJson, options)` - добавить гексагональный слой
- `removeHexagonLayer(layerId)` - удалить гексагональный слой
- `removeAllHexagonLayers()` - удалить все гексагональные слои
- `hasHexagonLayer(layerId)` - проверить наличие слоя
- `getHexagonLayersCount()` - получить количество слоев
- `updateHexagonLayerStyle(layerId, style)` - обновить стиль слоя

**Особенности**:
- Отдельное хранилище для гексагональных слоев (`hexagonLayers` Map)
- Поддержка hover эффектов
- Опциональные tooltips с H3 индексами
- Автоматическая стилизация

### 3. geo_area_map_controller.js (Stimulus)

**Расположение**: `assets/controllers/geo_area_map_controller.js`

**Новые targets**:
- `hexagonToggle` - переключатель гексагональной сетки
- `hexagonResolution` - слайдер разрешения
- `hexagonResolutionWrapper` - обертка контролов разрешения
- `resolutionLabel` - метка текущего разрешения
- `resolutionInfo` - информация о разрешении

**Новые методы**:
- `toggleHexagonGrid(event)` - переключение отображения сетки
- `onHexagonResolutionChange(event)` - изменение разрешения
- `_updateHexagonGrid()` - обновление сетки на карте
- `_setupMapEventListeners()` - настройка слушателей событий карты

**Особенности**:
- Автоматическое обновление сетки при изменении видимой области (moveend, zoomend)
- Ограничение количества гексагонов до 2000 для производительности
- Динамическое изменение разрешения через UI

## Разрешения H3

| Разрешение | Площадь гексагона | Размер стороны | Применение |
|------------|-------------------|----------------|------------|
| 5 | 252.90 км² | 8.54 км | Города |
| 6 | 36.13 км² | 3.23 км | Районы города |
| 7 | 5.16 км² | 1.22 км | Кварталы |
| 8 | 0.74 км² | 461.35 м | Улицы (по умолчанию) |
| 9 | 0.10 км² | 174.38 м | Здания |
| 10 | 0.015 км² | 65.91 м | Участки |

**По умолчанию используется разрешение 8** (оптимальный баланс между детализацией и производительностью).

## Производительность

### Оптимизации

1. **Ограничение количества гексагонов**: Максимум 2000 гексагонов отображается одновременно
2. **Ленивая генерация**: Гексагоны генерируются только для видимой области карты
3. **Отключение tooltips по умолчанию**: Для снижения нагрузки на рендеринг
4. **Автоматическое обновление при изменении viewport**: Гексагоны обновляются только при moveend/zoomend

### Рекомендации

- Для больших областей используйте меньшее разрешение (5-7)
- Для детальной работы с небольшими районами используйте большее разрешение (8-10)
- При медленной работе уменьшите разрешение или отключите сетку

## Использование

### На основной карте ServiceArea

1. Откройте ServiceArea в Sonata Admin
2. Перейдите на вкладку "Geo Areas"
3. Над картой включите переключатель "Показать гексагональную сетку"
4. Настройте детализацию с помощью слайдера

### В модальном окне создания кастомной зоны (интерактивный выбор)

1. На карте ServiceArea нажмите кнопку "Создать кастомную зону"
2. В открывшемся модальном окне над картой включите переключатель "Показать гексагональную сетку"
3. Настройте детализацию с помощью слайдера (5-10)
4. **Кликайте на гексагоны** для выбора (выбранные подсвечиваются зеленым)
5. Кликните повторно на гексагон чтобы отменить выбор
6. Наблюдайте счетчик выбранных гексагонов
7. При необходимости используйте кнопку "Очистить" для сброса всех выборов
8. Введите название и сохраните
9. **Выбранные гексагоны автоматически объединятся в один монолитный полигон** (dissolve operation)

**Результат**: Зона сохраняется как один Polygon с общими границами (или MultiPolygon если есть несколько отдельных областей)

**Альтернативный способ**: используйте стандартное рисование полигона инструментами Leaflet.draw (без гексагонов)

### Программное использование

```javascript
// Получить контроллер
const controller = this.application.getControllerForElementAndIdentifier(
    element,
    'geo-area-map'
);

// Включить гексагональную сетку программно
controller.hexagonGridEnabled = true;
controller._updateHexagonGrid();

// Изменить разрешение
controller.hexagonService.setResolution(9);
controller._updateHexagonGrid();
```

## Возможные проблемы и решения

### Проблема: Сетка не отображается

**Решение**: 
1. Проверьте, что переключатель включен
2. Проверьте консоль браузера на ошибки
3. Убедитесь, что h3-js установлен: `npm list h3-js`
4. Пересоберите assets: `npm run build`

### Проблема: Медленная работа

**Решение**:
1. Уменьшите разрешение (используйте 5-7 вместо 8-10)
2. Проверьте количество отображаемых гексагонов в консоли
3. Отключите hover эффекты в опциях `addHexagonLayer`

### Проблема: Гексагоны не обновляются при изменении карты

**Решение**:
1. Проверьте, что события карты подписаны в `_setupMapEventListeners`
2. Проверьте, что `hexagonGridEnabled === true`
