# Интерактивный выбор гексагонов для создания кастомных зон

## Новая функциональность

Добавлена возможность **создания кастомных гео-зон путем выбора гексагонов кликами** вместо ручного рисования полигонов.

## Преимущества

✅ **Проще и быстрее** - не нужно рисовать сложные полигоны
✅ **Точнее** - гексагоны H3 имеют стандартизированные размеры
✅ **Удобнее** - один клик = один гексагон
✅ **Гибкость** - можно комбинировать любое количество гексагонов
✅ **Визуальная обратная связь** - зеленый цвет показывает выбранные гексагоны
✅ **Автоматическое объединение** - гексагоны сливаются в монолитный полигон при сохранении
✅ **Чистая геометрия** - в БД сохраняется один Polygon без внутренних границ

## Как использовать

### Создание кастомной зоны из гексагонов

1. Откройте **ServiceArea** → вкладка **"Geo Areas"**
2. Нажмите кнопку **"Создать кастомную зону"**
3. В модальном окне включите **"Показать гексагональную сетку"**
4. **Настройте разрешение** слайдером (рекомендуется 7-8 для городов)
5. **Кликайте на гексагоны** для выбора:
   - Оранжевый гексагон = доступен для выбора
   - Зеленый гексагон = выбран
   - Клик на зеленый гексагон = отмена выбора
6. Наблюдайте **счетчик выбранных гексагонов** в левой панели
7. При необходимости используйте кнопку **"Очистить"** для сброса
8. Введите **название зоны**
9. Нажмите **"Сохранить"**

### Результат

При сохранении выбранные гексагоны автоматически объединяются в `MultiPolygon` и сохраняются в БД как кастомная гео-зона.

## UI элементы

### Панель статистики (появляется при выборе гексагонов)

- **Счетчик** - показывает количество выбранных гексагонов
- **Кнопка "Очистить"** - сброс всех выборов
- **Подсказка** - напоминает о клике для выбора/отмены

### Цветовая схема

| Цвет | Состояние | Действие |
|------|-----------|----------|
| Оранжевый (#ff6600) | Доступен для выбора | Клик = выбрать |
| Зеленый (#00ff00) | Выбран | Клик = отменить выбор |
| Светло-оранжевый | Hover над доступным | - |

## Технические детали

### Архитектура

Решение следует принципам **SOLID**:

1. **HexagonService** - конвертация H3 индексов в GeoJSON, определение гексагональных зон
2. **MapService** - управление слоями на карте (базовая сетка + выбранные)
3. **custom_area_modal_controller** - логика выбора, редактирования и сохранения

### Хранение данных

- **selectedHexagons** - Set<string> с H3 индексами (временно, только при создании)
- **selectedHexagonsLayerId** - отдельный слой для визуализации выбранных
- **currentGeometry** - Polygon/MultiPolygon GeoJSON после merge для сохранения

### Логика выбора

```javascript
// При клике на гексагон:
_toggleHexagonSelection(h3Index) {
    if (selectedHexagons.has(h3Index)) {
        // Убрать из выбранных
        selectedHexagons.delete(h3Index);
    } else {
        // Добавить в выбранные
        selectedHexagons.add(h3Index);
    }
    
    // Обновить визуализацию
    _updateSelectedHexagonsLayer();
    
    // Обновить геометрию
    _updateGeometryFromSelectedHexagons();
}
```

### Объединение гексагонов (Merge/Dissolve)

При сохранении выбранные гексагоны **объединяются в один монолитный полигон**:

```javascript
// 1. Конвертируем H3 индексы в Features
const features = h3Indexes.map(h3Index => 
    turf.polygon([hexagonService.hexagonToGeoJson(h3Index).coordinates])
);

// 2. Создаем FeatureCollection
const fc = turf.featureCollection(features);

// 3. Используем dissolve для объединения (убирает внутренние границы)
const dissolved = turf.dissolve(fc);

// 4. Результат -> один Polygon с внешним контуром
// Если гексагоны не все соседние -> MultiPolygon из нескольких областей
```

**Преимущества dissolve:**
- ✅ Убирает внутренние границы между соседними гексагонами
- ✅ Создает "гладкий" контур зоны
- ✅ Оптимизирует размер данных (меньше точек)
- ✅ Результат выглядит как нарисованный вручную полигон

## Совместимость

### Два режима создания зон

Система поддерживает **оба способа** создания кастомных зон:

1. **Интерактивный выбор гексагонов** (новый способ)
   - Включите гексагональную сетку
   - Кликайте на гексагоны
   - Система автоматически создаст MultiPolygon

2. **Ручное рисование полигона** (существующий способ)
   - Используйте инструменты Leaflet.draw
   - Рисуйте полигон вручную
   - Работает как раньше

Способы **не конфликтуют**: если есть выбранные гексагоны, они имеют приоритет.

## Рекомендации

### Выбор разрешения

| Задача | Рекомендуемое разрешение | Размер гексагона |
|--------|-------------------------|------------------|
| Большой район города | 6-7 | 1-3 км² |
| Несколько кварталов | 7-8 | 0.7-5 км² |
| Точный выбор улиц | 8-9 | 0.1-0.7 км² |
| Мелкие участки | 9-10 | 0.01-0.1 км² |

### Производительность

- Для больших зон используйте меньшее разрешение (6-7)
- Оптимальное количество гексагонов: **10-50**
- Максимальное на карте: **2000**

## Примеры использования

### Создание зоны доставки для района

1. Разрешение: **7** (кварталы ~5 км²)
2. Выберите 10-20 гексагонов покрывающих район
3. Название: "Центральный район"
4. Сохраните

### Создание зоны для нескольких улиц

1. Разрешение: **8** (улицы ~0.7 км²)
2. Выберите 5-15 гексагонов вдоль улиц
3. Название: "Улица Ленина и окрестности"
4. Сохраните

### Редактирование созданной зоны

Выбранные гексагоны сохраняются как **MultiPolygon** (набор отдельных гексагональных полигонов).

**При редактировании существующей зоны (созданной из гексагонов)**:

1. Откройте зону для редактирования
2. На карте отобразится **объединенный монолитный полигон** (синий)
3. Доступны **стандартные инструменты редактирования** Leaflet.draw (двигать вершины, менять форму)
4. **Если хотите добавить гексагоны** - включите чекбокс "Show hexagonal grid"
5. Выберите новые гексагоны кликами (станут зелеными)
6. При сохранении **все объединится в один монолит** через dissolve

**Важно**: После dissolve информация об исходных H3 индексах теряется. Восстановить их точно невозможно.

**Преимущества**:
- ✅ **Полный контроль** - добавляйте/удаляйте гексагоны при редактировании
- ✅ **Визуальная обратная связь** - видите все выбранные гексагоны
- ✅ **Гибкость** - можно расширить или уменьшить зону
- ✅ **Сохранение структуры** - гексагоны остаются отдельными полигонами
- ✅ **Точное редактирование** - можно вернуться и изменить любой гексагон

## Возможные проблемы

### Гексагоны не кликабельны

**Причина**: Гексагональная сетка не включена

**Решение**: Включите переключатель "Показать гексагональную сетку"

### При клике ничего не происходит

**Причина**: Возможно кликаете между гексагонами

**Решение**: Кликайте точно по центру гексагона

### Слишком много гексагонов

**Причина**: Высокое разрешение + большая область

**Решение**: Уменьшите разрешение до 6-7 или зумируйте карту
