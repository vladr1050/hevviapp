# Custom Geo Areas - Кастомные Географические Зоны

## Описание

Реализована функциональность создания и редактирования кастомных географических зон в ServiceArea через интерактивное рисование полигонов на карте.

## Архитектура

Решение следует принципам **SOLID** и использует идиомы **Symfony 8**, **Sonata Admin 4.4** и **Stimulus**.

### Backend (PHP)

#### 1. Entity: `GeoArea`
- Поле `scope` поддерживает три типа: `COUNTRY`, `CITY`, `CUSTOM_AREA` (значение 3)
- Кастомные зоны связываются со страной через `country_iso3`
- Геометрия хранится в PostGIS формате `MULTIPOLYGON`

#### 2. Repository: `GeoAreaRepository`
- Метод `findCustomAreasByCountryISO3()` - получение кастомных зон по стране

#### 3. API Controller: `GeoAreaController`
**Эндпоинты:**
- `GET /geo-area/custom-areas` - получение списка кастомных зон по country_iso3
- `POST /geo-area/custom-area/session` - сохранение кастомной зоны в сессию (временно)
- `GET /geo-area/custom-area/session/{id}` - получение кастомной зоны из сессии
- `DELETE /geo-area/custom-area/session/{id}` - удаление из сессии

#### 4. Service: `CustomGeoAreaService`
**Отвечает за:**
- Управление кастомными зонами в сессии
- Конвертацию GeoJSON в WKT для PostGIS
- Персистенцию из сессии в БД

**Методы:**
- `getFromSession()` - получить все кастомные зоны из сессии
- `persistFromSession($tempId)` - сохранить в БД из сессии
- `updateFromSession($geoArea, $tempId)` - обновить существующую зону
- `removeFromSession($tempId)` - удалить из сессии
- `clearSession()` - очистить все кастомные зоны из сессии

#### 5. Admin: `ServiceAreaAdmin`
**Хуки:**
- `prePersist()` - обработка кастомных зон при создании ServiceArea
- `preUpdate()` - обработка кастомных зон при редактировании ServiceArea
- `processCustomGeoAreas()` - единая логика обработки зон из сессии

## Workflow

### Создание ServiceArea с кастомной зоной

1. **Пользователь открывает форму создания ServiceArea**
2. **Выбирает страну** - загружаются города и кастомные зоны
3. **Нажимает "Создать кастомную зону"**
   - Открывается модальное окно
   - Пользователь вводит название
   - Рисует полигон на карте
   - Нажимает "Сохранить"
4. **Кастомная зона сохраняется в сессию**
   - Генерируется временный ID (temp_xxx)
   - Отображается на карте с бейджем "новая"
   - Кнопка меняется на "Редактировать кастомную зону"
5. **Пользователь нажимает "Create" или "Create and exit"**
   - ServiceAreaAdmin вызывает `prePersist()`
   - Все кастомные зоны из сессии сохраняются в БД
   - Связываются с ServiceArea
   - Сессия очищается

### Редактирование кастомной зоны

1. **Пользователь открывает форму редактирования ServiceArea**
2. **Если есть временная кастомная зона** - кнопка отображается как "Редактировать"
3. **Нажимает "Редактировать кастомную зону"**
   - Открывается модальное окно в режиме `edit`
   - Загружается существующая геометрия
   - Пользователь редактирует полигон
   - Нажимает "Сохранить"
4. **Обновленная зона сохраняется в сессию**
5. **При сохранении ServiceArea** - обновляется в БД

### Удаление кастомной зоны

1. **Пользователь нажимает кнопку "Удалить" у кастомной зоны**
2. **Зона удаляется с карты и из списка**
3. **Если это была временная зона:**
   - Удаляется из сессии
   - Кнопка меняется обратно на "Создать кастомную зону"

## Известные ограничения

1. **Один полигон за раз:** В модальном окне можно нарисовать только один полигон. При рисовании нового предыдущий удаляется.

2. **Только MultiPolygon:** Геометрия всегда конвертируется в MultiPolygon, даже если нарисован только один полигон.

3. **Временные зоны в сессии:** Кастомные зоны хранятся в сессии до сохранения ServiceArea. При закрытии браузера или истечении сессии они теряются.

## Troubleshooting

### Карта не отображается

**Проблема:** Карта показывает серый квадрат

**Решение:**
1. Проверьте консоль браузера на ошибки
2. Убедитесь, что Leaflet и Leaflet.draw установлены: `npm list leaflet leaflet-draw`
3. Пересоберите frontend: `npm run build`

### Leaflet.draw не работает

**Проблема:** Кнопки рисования не появляются

**Решение:**
1. Проверьте импорт в `custom_area_modal_controller.js`
2. Убедитесь, что CSS загружен: `import 'leaflet-draw/dist/leaflet.draw.css'`
3. Проверьте консоль на ошибку: `L.Control.Draw is undefined`

### Кастомная зона не сохраняется

**Проблема:** После создания ServiceArea кастомная зона не отображается

**Решение:**
1. Проверьте логи Symfony: `tail -f var/log/dev.log`
2. Убедитесь, что `CustomGeoAreaService` инжектирован в `ServiceAreaAdmin`
3. Проверьте данные в таблице `geo_area` с `scope = 3`

### Ошибка при конвертации GeoJSON в WKT

**Проблема:** Исключение при сохранении геометрии

**Решение:**
1. Проверьте формат GeoJSON в методе `convertGeoJsonToWkt()`
2. Убедитесь, что координаты имеют правильный порядок: [lng, lat]
3. Проверьте валидность полигона (первая и последняя точки должны совпадать)

## Примеры кода

### Создание кастомной зоны программно

```php
use App\Entity\GeoArea;
use App\Service\CustomGeoAreaService;

// В контроллере или сервисе
$geoArea = new GeoArea();
$geoArea->setName('Кастомная зона');
$geoArea->setScope(GeoArea::SCOPE['CUSTOM_AREA']);
$geoArea->setCountryISO3('LVA');

// GeoJSON геометрия
$geoJson = [
    'type' => 'MultiPolygon',
    'coordinates' => [[[[24.0, 56.9], [24.1, 56.9], [24.1, 57.0], [24.0, 57.0], [24.0, 56.9]]]]
];

// Конвертация в WKT
$wkt = $customGeoAreaService->convertGeoJsonToWkt($geoJson);
$geoArea->setGeometry($wkt);

$entityManager->persist($geoArea);
$entityManager->flush();
```

### Получение кастомных зон в JavaScript

```javascript
// В Stimulus контроллере
async loadCustomAreas(countryISO3) {
    const customAreas = await this.apiService.getCustomAreas(countryISO3);
    
    customAreas.forEach(area => {
        console.log(area.name, area.id);
    });
}
```
