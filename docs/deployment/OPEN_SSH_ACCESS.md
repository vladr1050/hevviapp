# Как открыть SSH на сервере 37.27.188.238

Два места, где нужно разрешить порт 22: панель Hetzner и сам сервер (если SSH там не установлен или закрыт UFW).

---

## Часть 1. Файрвол в панели Hetzner

1. Открой в браузере: **https://console.hetzner.cloud**
2. Войди в аккаунт и выбери **проект**, где создан сервер.
3. В левом меню нажми **Firewalls**.
4. **Если у сервера уже привязан файрвол:**
   - Нажми на этот файрвол.
   - Нажми **Edit** (или «Добавить правило»).
   - В блоке **Inbound** добавь правило:
     - **Protocol:** TCP  
     - **Port:** 22  
     - **Source:** `0.0.0.0/0` (или свой IP, если хочешь ограничить доступ)
   - Сохрани изменения (**Save**).
5. **Если файрвола нет или сервер к нему не привязан:**
   - Создай новый файрвол: **Add Firewall**.
   - Имя, например: `allow-ssh-http`.
   - В **Inbound** добавь два правила:
     - TCP, port **22**, source `0.0.0.0/0`
     - TCP, port **8090**, source `0.0.0.0/0` (для бэк-офиса)
   - В **Apply to** выбери свой сервер (37.27.188.238).
   - Сохрани.
6. Подожди 1–2 минуты и проверь с Mac:  
   `ssh root@37.27.188.238`

---

## Часть 2. Установка и включение SSH на сервере

Войти на сервер можно только через **консоль в браузере** (пока SSH недоступен).

### Вариант A: Обычная консоль (логин/пароль работают)

1. В Hetzner: **Servers** → выбери сервер → вкладка **Console**.
2. Нажми **Launch Console** (откроется окно с терминалом).
3. Введи логин (например `root` или `ubuntu`) и пароль.
4. В консоли выполни по порядку:

```bash
# Обновить список пакетов
sudo apt update

# Установить OpenSSH-сервер
sudo apt install -y openssh-server

# Включить автозапуск SSH при загрузке
sudo systemctl enable ssh

# Запустить SSH сейчас
sudo systemctl start ssh

# Проверить, что служба работает (должно быть "active (running)")
sudo systemctl status ssh
```

5. Открыть порт 22 в UFW (если файрвол включён):

```bash
sudo ufw allow 22/tcp
sudo ufw reload
sudo ufw status
```

6. Выйди из консоли. С Mac проверь:  
   `ssh root@37.27.188.238`

---

### Вариант B: Консоль не пускает — вход через Rescue

1. В панели Hetzner: **Servers** → твой сервер → вкладка **Rescue**.
2. Включи **Activate rescue system**, тип **Linux**.
3. Запомни или сохрани пароль root для Rescue (показывается на странице).
4. Нажми **Restart** у сервера и подожди 2–3 минуты.
5. Открой **Console** (Launch Console).
6. В консоли Rescue введи: логин **root**, пароль из шага 3.
7. В Rescue выполни (подставь свой диск, если не sda1 — смотри вывод `lsblk`):

```bash
# Посмотреть диски (корень обычно на sda1 или nvme0n1p1)
lsblk

# Смонтировать системный раздел (часто sda1)
mount /dev/sda1 /mnt

# Перейти в свою систему
chroot /mnt

# Установить SSH
apt update
apt install -y openssh-server
systemctl enable ssh

# Выйти из chroot
exit

# Размонтировать и перезагрузить
umount /mnt
reboot
```

8. После перезагрузки в панели Hetzner зайди снова в **Rescue** и **выключи** Rescue (Deactivate).
9. Сервер загрузится в обычную Ubuntu. Подожди 1–2 минуты.
10. Зайди в **Console** (уже обычная система), введи логин и пароль. Открой порт 22 в UFW:

```bash
sudo ufw allow 22/tcp
sudo ufw reload
```

11. Проверь с Mac:  
   `ssh root@37.27.188.238`

---

## Проверка с твоего Mac

В терминале:

```bash
# Доступность порта 22
nc -zv 37.27.188.238 22
```

Если видишь `succeeded` — порт открыт. Затем:

```bash
ssh root@37.27.188.238
```

Введи пароль. После первого входа рекомендуется сменить пароль:  
`passwd`

## Краткий чеклист

- [ ] В Hetzner Firewall: правило Inbound TCP 22 (и при необходимости 8090).
- [ ] На сервере: установлен `openssh-server`, служба `ssh` включена и запущена.
- [ ] На сервере: в UFW разрешён порт 22 (`ufw allow 22/tcp`, `ufw reload`).
- [ ] С Mac: `ssh root@37.27.188.238` подключается.
